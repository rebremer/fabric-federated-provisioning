Demo flow for scripts/provision_fabric.py — three federated personas, seven steps.

Personas:
  - Fabric admin: tenant admin, manages OPDG + VDG. One-time bootstrap only.
  - Platform SPN: member of both <platform-workspace-secgrp-name> and
    <platform-gateway-secgrp-name>. Creates the Fabric capacity (via ARM),
    workspaces, and federates access to team security groups. Never touches
    source/target data.
  - Team SPN: member of <team-secgrp-name>. Creates connections, pipelines, and
    runs them inside its assigned workspace.

Prerequisites (out-of-band, before running the script):
  - Platform SPN has Contributor (or Microsoft.Fabric/capacities/write) on the Azure RG
    named in capacity.resource_group. Step 2 self-assigns capacity Admin via
    properties.administration.members — no manual Fabric Admin portal step required
    (the per-capacity Contributors allow-list has no public REST API; this side-steps
    it by making the SPN a capacity Admin via ARM). If the role assignment is missing,
    step 2 fails fast and prints the exact `az role assignment create` command to fix it.
  - <team-secgrp-name> granted "Storage Blob Data Contributor" on the target ADLS Gen2 account.
  - All three secgrps exist; SPNs are members of their respective groups.

Note: the tenant allow-list "Service principals can create workspaces, connections, and
deployment pipelines" is managed by step 1.3 (mandatory) — list the exact setting name(s)
under `tenant_settings.enabled_setting_names` in YAML; discover with the `tenant-settings`
helper.

Steps:
  1. Fabric admin grants <platform-gateway-secgrp-name> Admin on OPDG + VDG (one-time per gateway).
  2. Platform SPN ARM-creates a Microsoft.Fabric/capacities F2 capacity and self-assigns
     capacity Admin. Persists the Fabric capacity GUID to <cfg>.state.yaml.
  3. Platform SPN creates the workspace (bound to the capacity from step 2) and grants on it:
       - <team-secgrp-name> Contributor
       - Team SPN direct Contributor (belt-and-braces for group propagation lag)
  4. Platform SPN grants <team-secgrp-name> ConnectionCreatorWithResharing on OPDG + VDG.
  5. Team SPN creates the SQL source connection (OPDG) + ShareableCloud ADLS Gen2 target.
  6. Team SPN creates the Fabric copy pipeline (SQL -> ADLS Gen2 Parquet).
  7. Team SPN runs the pipeline and polls until completion.


# --- Step 1: Fabric admin (one-time) ---
az logout
az login
python scripts/provision_fabric.py 1 config/prod-01.yaml

# --- Steps 2-4: Platform SPN ---
az logout
# NEVER paste the real secret here. Use $env:PLATFORM_SPN_SECRET, Get-Secret, or paste at the prompt only.
az login --service-principal --username <PLATFORM_SPN_APP_ID> --tenant <TENANT_ID> --password <PLATFORM_SPN_SECRET> --allow-no-subscriptions
python scripts/provision_fabric.py 2 config/prod-01.yaml   # Fabric capacity (ARM create + self-admin)
python scripts/provision_fabric.py 3 config/prod-01.yaml   # workspace + workspace RBAC
python scripts/provision_fabric.py 4 config/prod-01.yaml   # gateway federation (skip if no OPDG/VDG)

# --- Steps 5-7: Team SPN (member of <team-secgrp-name>) ---
az logout
# NEVER paste the real secret here. Use $env:TEAM_SPN_SECRET, Get-Secret, or paste at the prompt only.
az login --service-principal --username <TEAM_SPN_APP_ID> --tenant <TENANT_ID> --password <TEAM_SPN_SECRET> --allow-no-subscriptions

python scripts/provision_fabric.py 5 config/prod-01.yaml   # connections
python scripts/provision_fabric.py 6 config/prod-01.yaml   # pipeline
python scripts/provision_fabric.py 7 config/prod-01.yaml   # run + poll
